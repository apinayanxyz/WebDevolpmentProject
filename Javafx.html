Useful code

AnimationTimer

This code will allow you to do tasks that require no time limit, need to happen very fast, and need to run in the background, such as movement in games and replacing a label that displays information(lives, score, etc)

Shown below is an example of an animationTimer

private void GameLoop() {

new AnimationTimer() {

@Override

public void handle(long now) {

moveProjectile();

checkProjectileHit();

livesLabel.setText("Lives:" + player.getLives());

root.getChildren().remove(livesLabel);

root.getChildren().add(livesLabel);

if (player.getLives() == 0) {

stop();

gameTimeline.stop();

timerTimeline.stop();

AfterScreen afterScreen = new AfterScreen(SCREENWIDTH, SCREENHEIGHT, false, root, scene);

afterScreen.endGame();

}

if (timeSeconds == 30) {

stop();

gameTimeline.stop();

timerTimeline.stop();

AfterScreen afterScreen = new AfterScreen(SCREENWIDTH, SCREENHEIGHT, true, root, scene);

afterScreen.endGame();

}

}

}.start();

}

This can be called using the following.

GameLoop();

Keyframes

This code also works on the background, but works using cycles which run for a duration.

The number of cycles can be set to any number

The duration of each cycle can also be set to pretty much any number

Shown below is an example of a Timer created using keyframes

private void createTimerTimeline() {

timerTimeline.setCycleCount(THIRTY);

timerTimeline.getKeyFrames().add(new KeyFrame(Duration.seconds(1),

new EventHandler() {

@Override

public void handle(Event event) {

timeSeconds++;

timeLabel.setText("Time Remaining:" + (30 - timeSeconds));

root.getChildren().remove(timeLabel);

root.getChildren().add(timeLabel);

}

}));

}

To call this timer, the following code is used

timerTimeline.playFromStart();

This code can be used when a specific action needs to happen at a certain time(creating objects, a timer ,etc;)

Useful tips

When trying to create a JavaFX project, try to remember the following

  • Comment every sizeable chunk of code with block comments(/*Comment*/)
  • Split your code into subroutines with very clear names
  • keep your variable names short, memorable, easy to write or a combination of all three
  • Use class inheritance if you classes share many methods and varirables
